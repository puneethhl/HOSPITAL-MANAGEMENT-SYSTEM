1. Project Scope & Objectives

The core objective of a **Hospital Management System (HMS)** is to automate medical workflows, coordinate staff, track billing transactions, and maintain persistent medical history.

  • Demographics Tracking: Safely register patient parameters (DOB, contact details).
  • Staff Scheduling: Map doctors to specific medical divisions.
  • Appointment Registry: Manage date, time, and session status to avoid overlaps.
  • Clinical Logging: Log diagnostics, treatments, and prescriptions.

2. Relational Schema & Entities

A review of the 5 essential tables representing the database foundation.

Patients

Stores personal demography. Unique ID holds phone contact uniqueness constraint.

Doctors

Holds specialist rosters. Checks ensure salary field stays above zero.

Appointments

Associative junction table linking Patients and Doctors for consultations.

Medical Records

Persistent storage for diagnoses, medical prescriptions, and visit dates.

Billing

Tracks accounting, payments statuses, invoices, and totals.

3. Database Table Relationships

Relationship Primary Key (PK) Foreign Key (FK) Constraint / Action Rule
Patients (1) → Appointments (N) Patients.PatientID Appointments.PatientID ON DELETE CASCADE (Delete patient removes appointments)
Doctors (1) → Appointments (N) Doctors.DoctorID Appointments.DoctorID ON DELETE CASCADE (Delete doctor deletes appointment slots)
Patients (1) → MedicalRecords (N) Patients.PatientID MedicalRecords.PatientID ON DELETE CASCADE (Delete patient deletes history)
Appointments (1) → Billing (0..1) Appointments.AppointmentID Billing.AppointmentID ON DELETE SET NULL (Delete appointment leaves bill record intact)
Referential integrity checks enforce that no appointment is booked for a non-existent doctor or patient.

4. Crucial SQL Code Patterns

CREATE TABLE Appointments (
    AppointmentID INTEGER PRIMARY KEY AUTOINCREMENT,
    PatientID INTEGER NOT NULL,
    DoctorID INTEGER NOT NULL,
    AppointmentDate DATE NOT NULL,
    Status VARCHAR(20) DEFAULT 'Scheduled',
    FOREIGN KEY (PatientID) REFERENCES Patients(PatientID) ON DELETE CASCADE,
    FOREIGN KEY (DoctorID) REFERENCES Doctors(DoctorID) ON DELETE CASCADE
);
-- Adding New Patients
INSERT INTO Patients (FullName, Gender, DateOfBirth, Phone, Address)
VALUES ('John Doe', 'M', '1985-04-12', '555-1001', '123 Pine St, NY');

-- Scheduling Appointments
INSERT INTO Appointments (PatientID, DoctorID, AppointmentDate, AppointmentTime)
VALUES (1, 1, '2026-06-15', '09:00:00');
-- Inner Join to construct detailed consultations list
SELECT a.AppointmentID, p.FullName AS Patient, d.FullName AS Doctor, a.AppointmentDate
FROM Appointments a
JOIN Patients p ON a.PatientID = p.PatientID
JOIN Doctors d ON a.DoctorID = d.DoctorID;
-- Update billing to mark as paid
UPDATE Billing 
SET PaymentStatus = 'Paid' 
WHERE BillID = 3;

-- Cancel/Delete Appointment
DELETE FROM Appointments 
WHERE AppointmentID = 5;
